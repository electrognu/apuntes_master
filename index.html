Areas de trabajo y acciones

Areas

  • Working Area: files of the project in which we are working on.
  • Staging Area:Temporal zone where the modifed files that we need to add to our version control are waiting to be added to the repository..
  • Repository : files and logs on the repository

Acciones

  • Staging: Pass te files from the working area to stage area.
  • From the staging area to Repository we will do a commit.

Configuració git bàsica

git config --global user.name minombre

git config --global user.email email@dominio.es

git config --global core.editor "code --wait"

git config --global -e (editamos la configuración GIT)

git config --global core.autocrlf input - en windows substituir "input" por "true"

Utilizació bàsica

git status #te dice como está el seguimento de los ficheros dentro del repositorio donde esteimos situados

git status -s # silent

git add test.txt #agregamos el fichero test.txt al “stage area"

git commit -m “Ejemplo de commit del fichero pascual.txt”

git rm fitxer.txt # Borra el fichero y realitza el paso a la tapa "stage" directament sin tenir que hacer "git add"

git mv fitxer1 fitxer2 ## cambio de lugar o de nombre ...

Archivo gitignore : para ignorar archivos que no han de ser vistos ( claves )…

git log ( codi sha ) # Enseña que “commits” tienes y cuales son sus codigos SHA

git checkout ( codi sha ) #Mou al commit que te el numero sha

git checkout master # te mou a la branca principal

Retornar a un “commit” anterior:

git reset --hard <commit-hash>: Restablece el repositorio a un commit anterior y descarta todos los cambios.

git revert <commit-hash>: Crea un nuevo commit que deshace los cambios de un commit anterior, manteniendo el historial.

git checkout -b nombre-de-la-nueva-rama <commit-hash>: Crea una nueva rama desde un commit anterior.

Branches

Primer assegura-nos que no hi ha res per “comitejar”.

git status

git branch # ens diu on esteim…

git checkout -b rama git # rea una branca nova

Merge

Mesclar una feature a la branca Develop

git status -s # no hi ha de haver res a comitejar.

git checkout Develop # Canviam a la branca Develop a la qual que volem transmetre els canvis

git merge feature # merge

git branch -d feature # Esborrar la branca
